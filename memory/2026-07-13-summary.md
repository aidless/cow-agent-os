## 2. **ECS 部署"先 stat 再相信"是硬规则**

今天踩了 1 次:GitHub raw URL 偶发 14 字节 404(应该是 CDN 缓存),如果不 `stat -c%s`,`supervisor restart` 会起来一个**空 Flask**,看上去 RUNNING 但所有路由 404。

```bash
curl -sSL -o file ... && stat -c%s file  # 14 B = 重拉
```

> **认知**:**任何远端拉文件 → 立刻 stat 验证 → 然后才用**。不是"建议",是硬规则,已进 RULE.md。

## 3. **Mock URL 必须 fetch API,绝不 redirect**

Phase 7 Stripe mock 一开始我让前端 `window.location.href = mock_url`,**翻车**:mock_url 是 `/api/v2/.../mock_confirm`,浏览器 GET 路由 → 404。

正确路径:**JS 永远 fetch API,后端返回 JSON,前端处理**。绝不让浏览器 redirect 到 API 路径。

> **认知**:**任何"假接口"设计,前端协议必须是 fetch,不是 redirect**。这适用于所有 mock 场景(支付/邮件/短信/扫码)。
