### 修复要点(防再翻车)

**修复代码** — 在 `_c9_parse_figure_block` 把 caption 检测改手算 brace-depth:
```python
cap_idx = figure_text.find('\\caption{')
if cap_idx >= 0:
    body_start = cap_idx + len('\\caption{')
    depth = 1
    i = body_start
    while i < len(figure_text) and depth > 0:
        ch = figure_text[i]
        # Skip $...$ and $$...$$ math so braces inside math
        # ($N{=}1$, $\Gamma_{\mathrm{...}}$, etc.) don't affect
        # the depth counter.
        if ch == '$':
            ...
        if ch == '{':
            depth += 1
        elif ch == '}':
            depth -= 1
            if depth == 0:
                break
        i += 1
```

**关键 3 点**:
1. 字符级 brace depth(不靠 regex 近似)
2. 跳过 `$...$` 和 `$$...$$` 数学模式(避免 math 内 `{}` 影响)
3. `depth == 0` 才算 balanced,否则保留 `has_caption=False`
