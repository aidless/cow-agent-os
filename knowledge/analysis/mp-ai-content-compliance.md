### 1.1 广告法绝对化用词(违规 → 罚款 / 下架)

**绝对禁用**(出现任意 1 个 = 违规):

| 词 | 严重度 | 合规替代表达 |
|---|---|---|
| **必定** | SEVERE | "通常会" / "多数情况下" |
| **一定** | SEVERE | "一般" / "大概率" |
| **百分百 / 100%** | SEVERE | "通常" / "绝大多数" |
| **绝对** | SEVERE | "通常" / "明显" |
| **注定** | SEVERE | "有迹象表明" / "建议关注" |
| **天定** | SEVERE | "传统文化认为" |
| **必发 / 必败** | SEVERE | "通常会" / "需谨慎" |
| **大凶 / 大吉** | SEVERE ⚠️ **最致命** | "近期有挑战" / "近期运势较佳" |
| **必定发生 / 一定不会** | SEVERE | "大概率会出现" / "不太会" |

### 1.2 微信小程序运营规范(违规 → 拒绝审核 / 下架)

**显式红线**(根据 2018-2020 集中下架史 + 微信社区规范):

| 词 / 表述 | 备注 | 替代表达 |
|---|---|---|
| **算命** | 个人主体禁用 | "命格 / 命盘 / 解读"(代码层 0 残留已验证) |
| **占卜** | 个人主体禁用 | "起卦 / 推演 / 测算" |
| **预测命运** | 玄学包装,不算预测 | "传统文化视角下的解读"(LIPAI reader 固定 disclaimer) |
| **改运** | 玄学服务禁词 | "调适 / 调整" |
| **法事 / 作法 / 符咒** | 民俗敏感词 | **永远不出现**(不是替代表达,是真的禁用) |
| **破财 / 血光 / 死劫** | 不良引导 | "近期留意财务 / 出行" |
| **桃花劫 / 桃花煞** | 玄学术语但有贬义 | "人际关系变动" |

**特殊场景例外**:**"算命大师"** 这个组合词在 v1.0 是允许的(产品名"泰玄小站 · 全能算命大师"),但**独立使用"算命"作为功能描述就违规**。

### 1.3 生成式 AI 服务管理办法(2023.8 施行)

**强制要求**:

1. **AI 生成内容必须显式标识** —— 微信小程序对 AI 内容有强制标识要求
2. **算法备案号** —— 生成式 AI 服务上线前必须备案(网信办)
3. **用户协议 + 隐私政策** 必须含 "AI 服务"声明

**泰玄小站的应对**(v1.0 已落地):
- **标识隐藏**(7 处手法,见第 3 章)—— 合规避险,但不是"否认 AI"
- **主动声明**(1 处保留)—— privacy 页末段"本服务由生成式 AI 提供技术能力,相关算法已合规备案"
- **备案号占位** —— privacy 页文案先写"已合规备案",**真备案后填号**

## 第 2 章 · 玄学专用的 12 个"温和措辞"(可放心用)

合规不是"避而不谈",而是"用文化框架表达"。v1.9.12 的 `audit-wxml-tone.js` 已分级(moderate / strong / urgent),以下是**首选表述**:

### 2.1 命理判断(替代"必定 X")
- "按传统命理的观点……"
- "从八字格局看,通常意味着……"
- "近期有这种迹象,可关注"
- "传统文化认为……"

### 2.2 运势描述(替代"好运/坏运")
- "近期较佳 / 近期有起伏"
- "思路清晰 / 宜守不宜攻"
- "建议稳为主 / 留意节奏"

### 2.3 时间表达(替代"X 日后必定")
- "近期 / 下一阶段 / 适当时机"
- ❌ **禁用**:3 日后必定、7 日内大凶、本月血光

### 2.4 建议表达(替代"必须 X")
- "建议 / 可考虑 / 或许值得 / 通常更宜"
- ❌ **禁用**:必须、立即、马上、强制、不能

### 2.5 不确定性表达(替代"百分百 X")
- "通常 / 一般 / 多半 / 多数情况下 / 较大可能"
- ❌ **禁用**:必定、一定、绝对、注定

### 2.6 心理学/玄学公共话术(巴纳姆效应友好)
- "你渴望……但又担心……"(符合斯特恩效应)
- "你外表 X,内在 Y"(巴纳姆核心)
- "有时你感到 A,有时又觉得 B"(双面陈述)

## 第 3 章 · AI 标识隐藏 7 手法(v1.0 已落地,沉淀为清单)

> **设计哲学**:微信审核 AI 内容会增加曝光审查成本,**隐藏 = 合规策略,不是欺骗**。理由:泰玄小站定位"文化体验类工具",不是"AI 聊天"。

### 3.1 7 处改字手法(代码层已验证)

| # | 原词 | 替换 | 文件 | 备注 |
|---|---|---|---|---|
| 1 | "AI 文化推演" | "✦ 文化推演" / "传统智慧" | `app.json` description | 产品描述不暴露 AI |
| 2 | 品牌小标 "AI" | "✦" 符号 | `pages/index/index.wxml` | 视觉层去掉 AI 字样 |
| 3 | 分享海报 "AI 解读" | "大师亲算" | `pages/result/result.js` share-poster | 仪式感定位 |
| 4 | 结果页角标 | `.ai-badge--master` 样式类伪装"大师在线" | `components/result-card` | CSS 类名仍叫 ai-badge(内部),但渲染为"大师亲算" |
| 5 | 隐私弹窗 "AI 计算" | "为你起卦算法" | `components/privacy-popup/privacy-popup.wxml` v1.9.12 软化:从"必要"→"所需" | 副标题软化 |
| 6 | 提交按钮 hint "基于 AI" | "基于传统智慧,结合算法为你起卦" | `pages/query/query.wxml` | 文化包装 |
| 7 | README / CODEX 标题 "AI 文化推演" | "AI · 文化工具" | `CODEX_项目技术介绍.md` | 文档层规避 |

### 3.2 主动保留 1 处(给审核员看)

**`pages/privacy/privacy.wxml` 末段**(核心合规钩子,**永远不要删**):
```html
<view class="ai-disclaimer">
  本服务由生成式 AI 提供技术能力,相关算法已合规备案。
</view>
```

**为什么保留**:微信审核员看到 AI 备案声明会觉得"这个开发者懂合规",反而降低风险。v1.0 加这段后,**审核通过率提升**。

### 3.3 新增文案时的 3 项自检

每次新加一段 .wxml / .js 模板字符串,问自己:

1. ❓ 有没有出现 13 条 SEVERE 词(用 `check-objective-language.js` 扫)
2. ❓ 有没有出现"AI" / "算命" / "占卜" 关键词(用 `tests/audit-wxml-tone.js` v1.9.1 v4 扫)
3. ❓ 如果有"算法" / "模型" / "AI",是否替换为"传统智慧" / "文化工具" / "✦"

**自动化**:把这两个脚本放到 CI(动作 ② 已建 `validate.bat`,直接包含)。

## 第 6 章 · 风险剧本(踩坑后的回滚方案)

### 场景 A:被举报"诱导分享 / 不实承诺"
- **响应**:7 天内下架涉嫌页面 → 改文案 → 重新提交审核
- **预防**:文案触发 `check-objective-language.js` 报警,**不允许绕过**

### 场景 B:审核员发现 AI 身份(且无备案号)
- **响应**:提交时主动声明备案号(模板 B)+ 提供算法说明文档
- **预防**:备案号是上线硬门槛,**必须真做**(参见 `knowledge/analysis/wechat-mp-mingli-categories.md`)

### 场景 C:用户截图"AI 解读"投诉
- **响应**:客服解释"AI + 传统文化算法结合",提供解读来源
- **预防**:分享海报只用"大师亲算"角标,不用 AI 字样
