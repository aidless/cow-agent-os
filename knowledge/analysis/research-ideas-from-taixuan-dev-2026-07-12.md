# 研究 Idea 挖掘:taixuan-web 部署实战(2026-07-12)

_从今日 taixuan-web v1.0/v1.1/v1.2 部署实战中提炼的 ML / 系统研究 idea 候选。_

## 一、Idea 列表

### Idea T1 · LLM 实时输出对用户感知延迟的影响量化

**核心问题**:LLM 生成 1000 token 实际耗时 12 秒。用户看到的"等待 15 秒"vs"逐字打字 15 秒"心理感受差多少?

**可测量**:
- TTFT (time-to-first-token)
- 平均 token / 秒速率
- 用户停留时长 vs 一次性返回
- 转化率(完成解读后是否再次访问)

**实验设计**:
- A/B 测试:同一用户随机收到流式 vs 一次性
- 测量指标:NPS / 重复访问 / 解读后分享

**新意**:**流式是行业默认**(ChatGPT / Claude / DeepSeek 全部流式),但**缺乏学术量化**。到底好多少?

**目标会议**:CHI / CSCW / IUI(人机交互)

**工作量**:2-3 月(需要真实流量 + 用户数据)

### Idea T2 · 小规模 LLM Web 应用的工程 pitfall 实证

**核心问题**:研究员写 ML paper 时普遍不写 production code。今日踩 6 个 bug,有什么共性?

**发现的 6 类 pitfall**:
1. **Python 装饰器错位**(route 装饰错对象)
2. **UTF-8 BOM 看不见的 SyntaxError**
3. **模块级 init 顺序错误**(`init_db` 在 log 之前)
4. **闭包状态污染**(generator + nonlocal 复杂)
5. **限流状态共享**(内存字典 vs Redis)
6. **CSP / HSTS 等安全 header 缺失**

**可输出**:
- **实证调查**:从 GitHub 上 100 个 LLM 应用 repo(类似 chatbot-UI / LibreChat / Open WebUI),自动化扫描这 6 类 pitfall 的出现频率
- **工具**:静态分析脚本 + CI 集成
- **建议清单**:ML 研究员做 web 部署的 10 条强制规范

**新意**:LLM 应用井喷,security audit 缺位

**目标会议**:NeurIPS Workshop / ICML Workshop(MLSys)

**工作量**:3-4 月(需要写工具 + 实证)

### Idea T3 · AI 应用的用户认证设计权衡(以文化类 LLM 为例)

**核心问题**:LLM 应用什么时候值得做用户系统?不做 vs 邮箱密码 vs OAuth 怎么选?

**v2.0 RFC 已写**(知识库 analysis/taixuan-web-v20-user-system-rfc-2026-07-12.md):

| 维度 | 不做 | 邮箱密码 | OAuth |
|---|---|---|---|
| 隐私 | 最好 | 中 | 差 |
| 工程量 | 0 | 14h | 30h |
| 用户体验 | 无登录 | 简单 | 一键登录但注册难 |
| 法律责任 | 最小 | 中 | 中 |

**可实证**:50 个 LLM 应用(chatbot / AI 工具 / 文化类)分析各自的认证策略,统计转化率 / 用户粘性差异。

**新意**:LLM 应用决策树 vs 传统 SaaS 完全不同(隐私要求 + 短期消费场景 + 低频使用)

**目标会议**:HCI / CHI Late-Breaking Work

**工作量**:1-2 月(实证 + 调研为主)

---

### Idea T4 · LLM 推理应用的限流与成本控制实证

**核心问题**:每分钟 10 次限流是凭直觉,有没有理论依据?

**今日设计**:内存 dict,单 IP 每 60s 限 N 次

**待研究问题**:
- **N = ?** 经验值?理论值?
- **per-IP vs per-user vs per-API-key?** 三种粒度差异?
- **rate-limit 后 UX?** 直接 429 错 vs 排队 vs 降级?
- **攻击者视角**:绕过限流的成本(代理池 / 慢速攻击)
- **诚实用户视角**:误限流的概率

**可做**:
- 50 个 LLM 应用公开的 rate limit 政策调查
- 模拟攻击 + 模拟诚实用户,找最优点
- 公式化:cost(LLM 调用) × P(误限) + P(漏限) × cost(被攻击)

**新意**:**rate limit 业界无理论**,全是经验值

**目标会议**:Security Workshop / MLSys

**工作量**:2 月(实证 + 模型)

---

### Idea T5 · 流式 SSE 与 AbortController 在 AI 推理 UX 的最佳实践

**核心问题**:流式输出是不是所有 LLM 场景都好?有没有反例?

**今日实现**:8 派 + abort 按钮 + AbortController

**待研究**:
- **什么场景流式反而差?**(短答案、JSON 结构化输出、tool calls)
- **abort 按钮的 UX 副作用**:用户更容易中断 = 数据质量差?
- **客户端 vs 服务端 abort**:浏览器刷新 / 用户点 X / 超时取消 三种路径
- **断点续传**是否值得做?

**可做**:
- A/B 测试:短 vs 长场景流式
- 收集 abort 事件的统计(多少是用户主动 / 多少是超时)

**新意**:UX 角度的细致量化

**目标会议**:IUI / CHI EA

**工作量**:2-3 月(需要 UX 实验设计)
