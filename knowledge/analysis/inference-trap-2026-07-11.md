1. **记忆层失败**:MEMORY.md / `memory/2026-07-10.md` 的"修到 v1.0.2"是**历史快照**,不是**当前 ground truth**。AI 跨 session 把历史当现状用,默认错。

## 🛡️ 应该遵守的硬规则(给所有 agent / 未来我自己)

### Rule 1:Read Before Touch

> 任何会话涉及具体文件,**第一次提到这个文件时,先 `read` 看 5-20 行**。

适用范围:
- "我要改 X 文件" → 先 read
- "X 文件不存在,我来写" → 先 `dir` 验证
- "X 文件是哪个版本" → 先 `read` 看 header

不适用:
- 用户明确说"新建"(`write` 本来就 create)
- 用户明确说"覆盖 X"(读现状无意义)

### Rule 2:Verify Existence

> 任何"X 存在吗"判断必须用 `ls` / `dir` 验证,**绝不**用记忆里"我记得存在 / 不存在"。

适用范围:
- "上次写了 X,这次接着改" → 必须 dir
- 跨 session 接力工作前 → 必须 dir
- "X 在哪里" → 哪个目录就 dir 哪个

### Rule 3:Two-Strike Reset

> 同一个问题连续 2 轮没解决,**强制**回到 read 现状重起炉灶。

机制:
- 1 轮没解决 → 假设还很健康,继续修
- 2 轮没解决 → 假设层可能就有问题,**强制**重新 read 现状
- 3 轮没解决 → 必须停下来跟用户对话复盘,而不是继续猜根因

今晚我违反这条最多 —— 5 轮了还在加修法。

### Rule 4:History is Hint, Not Ground Truth

> MEMORY.md / memory/*.md / 历史对话都是**提示**,不是 ground truth。每次任务确认现状只信当下 read 的内容。

适用范围:
- "上次做了 X" → 当提示,不直接复用为"现在 X 已完成"
- "上次踩过 X 坑" → 当提示,**仍然验证 X 是不是同一个坑**
- "上次这是 vX.Y" → 当提示,仍然 read 看版本

---

## 🎯 触发这规则的 5 类场景清单

下次会话开局,**只要触发下面任一场景**,就强制 read + dir:

1. **"接着上次的活"** → 必 read 现状文件
2. **"修这个 bug"** → 必 read bug 所在文件 + 依赖
3. **"X 文件不存在,新建"** → 必 dir 验证
4. **"按 vX.Y 模板来"** → 必 read 模板现状版本
5. **"我以为 X 是这样"** → 必 dir 拿真名单
